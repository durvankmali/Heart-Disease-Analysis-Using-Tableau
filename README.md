# Heart Disease Analysis: Identifying Clinical and Lifestyle Risk Factors with Tableau

## Project Name : **Heart Disease Analysis using Tableau**


## Project Overview

In the healthcare sector, clinicians often struggle to identify the cumulative impact of multiple lifestyle choices on cardiovascular health. This project delivers an **Interactive Clinical Analytics Dashboard** using **Tableau** and **MySQL**. By processing over **4,500 patient records**, this system uncovers hidden correlations—such as the compounding effect of smoking and alcohol—and introduces a custom **Patient Risk Score** to empower healthcare providers to prioritize high-risk patients effectively.


## Technology Stack

### Data Analytics and Visualization : 
**Tableau Desktop / Tableau Public**

### Database Management : 
**MySQL Workbench** (Used for Data Transformation and View Creation)

### Data Preparation : 
**SQL** (Utilizing CASE logic for BMI and Lifestyle grouping)

### Documentation : 
**Technical Templates** (Including DFD, Empathy Maps, and Solution Requirements)


## Project Phases

This repository contains all documentation and logic required for the project evaluation, organized by phase:

* **Ideation Phase**: Problem Statements for clinicians, Empathy Maps, and Idea Prioritization.
* **Project Design Phase I & II**: Customer Journey Maps, Data Flow Diagrams (DFDs), and 2-Tier Architecture.
* **Project Planning Phase**: Agile Sprint schedules, User Stories, and Velocity tracking.
* **Project Development Phase**: SQL View scripts, Tableau Calculation logic (Risk Scores), and Performance Testing results.
* **Final Deliverables**: The comprehensive project report and the 5-minute video demonstration link.


## How to Explore the Project

### 1. Database Setup

Import the heart disease dataset into **MySQL** and execute the provided View scripts to generate clinical categories.

### 2. Tableau Visualization

Open the **.twbx** file to access the **10 unique visualizations**, including Area Charts for age trends and Racewise Treemaps.

### 3. Live Dashboard

Access the finalized, high-performance dashboard via the **Tableau Public** link provided in the documentation.

### 4. Performance Check

Review the **Utilization of Filters** (Context and Extract) used to maintain responsiveness across the dataset.
